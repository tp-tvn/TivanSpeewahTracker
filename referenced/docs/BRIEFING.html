📋Executive Summary

Tivan Dashboard is a Streamlit-based activity tracking and cost management system for drilling projects. It integrates daily drilling operations (PLODs) with project budgets, timelines, and financial reporting to give real-time visibility into project health.

At a Glance
  • Purpose: Track drilling activities, manage budgets, forecast costs, and monitor schedule performance
  • Users: Project managers, drilling supervisors, financial analysts, site coordinators
  • Key Data: PLODs (daily drilling reports), budget allocations, hole purposes, cost rates
  • Main Outputs: Budget vs Actual reports, Gantt timelines, activity summaries, cost forecasts

🎯Core Capabilities

1. PLOD Ingestion & Validation

Import daily drilling reports (PLODs) from PDF or manual entry. The system extracts hole names, depths, drilling intervals, and equipment used. It validates data against known holes, flags anomalies, calculates costs automatically based on configured rates, and stores drilling intervals for historical analysis.

2. Budget Planning & Gantt Integration

Connect your project schedule to drilling budgets by importing Gantt charts as CSV/Excel, mapping Gantt items to drilling purposes, and setting budget amounts and date ranges. The system supports multiple phases per purpose (different contractors) and allows work group assignment to each budget period.

3. Budget vs Actual Reporting

Compare planned vs actual spending in real time. Track metres drilled vs planned metres, actual cost vs budgeted amount (calculated from PLOD data), cost per metre trending, and get variance alerts when overspending or ahead of schedule. Filter by purpose, rig, or date range.

4. Timeline Analysis (Gantt Charts)

Visualize planned vs actual project timelines with horizontal bar charts showing planned vs actual drilling dates. Identify schedule delays and early finishes, track progress by purpose or work group, and see which phases overlap.

5. Historical Analysis & Data Explorer

Dive deep into drilling data by searching PLODs by hole name, purpose, date, or rig. View drilling interval details, compare performance across holes and rigs, and identify patterns.

6. Financial Analytics

Understand costs and forecasts including cost per metre by drilling type, equipment and consumable expense tracking, labour allocation, cash flow forecasts, and variance analysis.

7. Admin Features (Password Protected)

Configure your system with password-protected admin panel: set drillhole purposes, manage budget rates, import/edit Gantt charts, assign work groups, configure rigs, set weather data, and correct OCR errors.

🚀Quick Start Guide (QRG)

First Run: One-Time Setup

Step 1: Define Your Drilling Purposes

Location: Admin Panel → 🎯 Drillhole Purposes

  1. Click "🔒 Admin" in top right, enter admin password
  2. Click "🎯 Drillhole Purposes" tab
  3. Import CSV: Prepare CSV with 2 columns (Hole Name, Purpose) → Click Upload → Import
  4. Or Manual Entry: Scroll to bottom and click table cells to add/edit

Step 2: Set Budget Rates

Location: Admin → 💰 Rate Management

  1. Set cost rates for each drill type + hole type combo
  2. Example: RC drilling × RCRD holes = $150/metre
  3. Mark "N/A" for combinations you don't use
  4. Save

Step 3: Configure Your Rigs

Location: Admin → 🔩 Rigs

  1. Add rig names (e.g., "RC-001", "Diamond Rig A")
  2. Set default drill type and hourly rates

Step 4: Import Your Gantt Chart (Budget Planning)

Location: Admin → 📊 Budget Targets

  1. Export your project Gantt chart as CSV or Excel
  2. Scroll to "📊 Import from Gantt Chart"
  3. Upload file and map Gantt items to purposes
  4. Assign Work Groups (contractor names)
  5. Click "Import Gantt Chart Allocations"

Daily Operations: Importing PLODs

Import a PLOD

Location: Main Dashboard → "Import PLOD"

Method 1: Upload PDF
Click "Import PLOD" → Select PDF file → Review extracted data → Click "Save PLOD"
Method 2: Manual Entry
Click "Import PLOD" → "Manual Entry" → Fill in form with date, rig, hole name, drilling intervals → Save

📊Daily Monitoring: Budget vs Actual

Location: Main Dashboard → "📊 Budget vs Actual"

This is your one-page health check. Key columns:

Column What It Means Action
Status 🟢 On Track / 🟡 Warning / 🔴 Risk Monitor if warning/risk
Actual Metres Total metres drilled so far Compare to Planned
Actual Cost Money spent so far Compare to Budget
Variance Difference from plan (%, $) < 0% = under budget ✓
Cost/Metre Actual cost per metre drilled Track efficiency

Interpreting Status

  • 🟢 Green (On Track): Within 10% of budget
  • 🟡 Yellow (Warning): Approaching limit or running behind schedule
  • 🔴 Red (Risk): Over budget or significantly behind schedule
If you see red: Click the purpose to view details → Check recent PLODs → Review actual rates vs configured rates → Escalate if overspend is expected

📅Schedule Tracking: Gantt Chart Tab

Location: Main Dashboard → "📅 Gantt Chart"

This shows a visual comparison of planned vs actual project timelines:

  • Blue bars = Planned timeline (from your Gantt import)
  • Green bars = Actual drilling (from PLOD data)

What to Look For

  • ✓ Green starts near blue = On schedule
  • ⚠️ Green starts after blue = Running late
  • ⚠️ Green much shorter than blue = Finished early
  • ? No green bar = Not yet started

❓Frequently Asked Questions

Setup & Configuration

Q: How do I know if my Gantt import worked?

A: Go to Admin → 📊 Budget Targets → scroll down to "Purpose Budget Allocations" table. You should see new rows with dates, budgets, and work groups.

Q: Can I have multiple budget periods for the same purpose?

A: Yes! Example: "2026 RC" from Jan-Mar (Contractor A), then Apr-Jun (Contractor B). Each period is a separate row in the budget allocations table.

Q: Can I edit a PLOD after importing it?

A: Go to Admin → 🗑️ Delete PLODs, find your PLOD and delete it, then re-import with corrected data.

Budget & Cost Reporting

Q: Why does my "Actual Cost" show $0 even though I imported PLODs?

A: Missing budget rate. Go to Admin → 💰 Rate Management → Budget Targets and check if the drill_type + hole_type combination has a rate. If blank, add a rate (e.g., $150/m).

Q: How is the actual cost calculated?

A: Actual Cost = Sum of (metres drilled × rate for that drill type)

Q: When should I start worrying about overspend?

A:

  • 80% of budget: Start monitoring closely
  • 90% of budget: Escalate to project manager
  • 100% of budget: Stop work until approved adjustment

Timeline & Gantt Chart

Q: Why doesn't my Gantt chart show any actual drilling data?

A: No PLODs have been imported yet, or they're for different purposes. Check Budget vs Actual tab to confirm PLODs imported. Ensure hole names match hole_purposes table exactly.

Q: Can I edit dates directly in the Gantt chart?

A: No. Edit in Admin → 📊 Budget → edit the start_date and end_date columns in the budget allocations table.

Troubleshooting

Q: Dashboard is slow or won't load

A: Refresh the page (Ctrl+R) → Clear browser cache → Check if large PLOD import is running → Contact admin if persists

Q: PLOD PDF import failed with "Unable to extract"

A: PDF might be image-only (no text layer). Use manual entry instead, or run PDF through OCR software first.

💡Pro Tips

  1. Use work groups for accountability: Assign each contractor to their phases. Easy to track who's responsible.
  2. Import Gantt chart early: Even rough dates are better than defaults. Gantt becomes your schedule baseline.
  3. Set realistic budget rates: Consult with procurement/finance on $/metre before drilling starts.
  4. Check dashboard weekly: Budget vs Actual should be reviewed at least weekly. Catch variances early.
  5. Keep hole names consistent: Use exact spelling—"TIVAN-001" won't match "Tivan-001".
  6. Export monthly snapshots: Download tables as CSV at end of month for historical record.
  7. Use Data Explorer for detailed dives: Budget vs Actual gives summary; Data Explorer shows details.
  8. Document manual edits: If you override a budget or delete a PLOD, note it somewhere for audit purposes.
